## 正文

用了快两年 ChatGPT / DeepSeek 写东西，我最烦的一件事是：**改一篇文章，要在对话窗口里反复复制粘贴**。

润色完粘回来，再框一段去扩写，再粘回来；一会儿又想让 AI 点评一下结构，又得重新贴一遍。一篇文章被我切得七零八落，最后连「到底哪一版是对的」都忘了。

于是前阵子我自己写了一个小工具：一块**画布**，左边放你的原文段落，右边接一个个 AI 操作框（润色、扩写、缩写、概括、点评、续写、翻译），拖一根线就连起来。点一下「向下游全部运行」，整篇自动跑完。

它不是又一个聊天机器人。它的核心是一个**「写作流水线」**：

- 你的每段文字是一个节点，AI 怎么处理它是另一个节点；
- 一个节点的输出能同时喂给多个下游，也能把多个上游拼起来一起处理（比如把「原文 + 大纲」一起丢给扩写）；
- 跑完的每一版结果自动存进「运行历史」，随时回看；
- 还能把整条链复制，原样套到另一篇文章上。

对写小说、博客、论文的人来说，真正累的从来不是「让 AI 写一句」，而是「在多个修改目标之间反复横跳」。这个工具把那张关系图固化下来，跑一次就全有了。

技术上它**完全跑在浏览器里**：你填自己的 API Key（默认 DeepSeek，也兼容 OpenAI 格式），文章不经过任何服务器，关掉网页数据就留在本地。不用注册、不收钱、没有任何请求发到我的后台——一个纯前端的玩具，但真的好用。

最近加了文档导入：直接丢一个 `.txt / .md / .docx / .pdf` 进去，按段落拆成节点，批量润色或翻译。科研论文也能整篇翻（顺带说明：公式类内容在 PDF 里多是图片、文本层提取会丢，这点我在工具里写清楚了，不是 bug）。

老实说现在还是个**早期框架**，很多我想做的还没落地：模板库、按小说 / 论文预设的专用节点、Markdown 渲染、分享链接……但核心的「画布 + 连线 + 批量跑」已经能用了。

工具开源在 GitHub，也能直接在网页里试用。如果你也烦「复制粘贴式写作」，欢迎来玩、来提意见——独立开发，想法比 star 重要：

## 附：为什么这么写（发布策略备注，发布时删掉本段）

- 知乎用户反感硬广，所以用「我自己也烦这个问题 → 顺手做了个工具」的第一人称叙事，比「快来用我家产品」自然得多。
- 强调「零后端 / 你自己的 Key / 不过服务器」是隐私卖点，也是和商业化 SaaS 的区隔，知乎技术读者吃这套。
- 主动承认「还是早期框架」反而增加可信度（indie hacker 人设），但马上接「核心能用」，不显得半成品。
- 结尾抛问题引导评论，知乎算法偏爱高互动。
- 配图建议：一张「原文段落 → 润色 → 扩写」的画布截图，比纯文字转化高。
